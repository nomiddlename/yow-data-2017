How ad serving works

Ad Breaches

It's that last class of ads that cause the problems. They're not going to be relevant to a lot of the audience, and in the worst case they could be offensive or harmful to the site's brand. An example would be a Ford ad when Toyota is your sponsor, or an advert for alcohol or gambling on a website for kids.

Now

  • Complaint
  • Panic
  • Fixed
Sites provide a list of categories to exclude. Doesn't always work, it relies on the people setting up the ads to use the right categories. Usually the first we will hear of an ad breach is a complaint, or an embarrassing social media post. At best it will be an email from an internal user. The next step in the process is for lots of people to panic. Once we get to this stage fixing it is quite a quick process. But it's the finding out part that we want to improve.

Keywords

We set up a list of keywords that would trigger an alert if any of them were found in the response from the Cloud Vision API.

Cloud Vision Advantages

  • managed
  • expertise
  • easy
There's no servers to set up or look after. Same technology that google uses in safe search, run by people much smarter than me. Probably smarter than you, too. It's easy - a http request and you're done.

Cloud Vision Disadvantages

  • Can't be used offline

Tensorflow

Tensorflow - open source machine learning library. Backed by Google, other libraries are out there. Keras can use tensorflow as a backend. Comes with support for a few different image recognition models, but we're going to use the best of them.

Inception V3

Here's a diagram which shows you how the model works. Simple, huh? Each of those blocks is layer in the neural network, with different numbers of nodes. So there's a whole lot of coefficients that get tweaked during training, and at the end these two layers here boil them down to classifications for the images.

Accuracy

The TF tutorial says you should be able to get 90-95% accuracy with only a hundred or so images per category. Which is great, but it means well-defined categories (the example is different kinds of flowers). I wanted to know how well it would perform on more ambiguous categories - I want breach or not-breach.

Images

I had access to a lot of old ad images, but not breaching images. So rather than go searching the naughtier parts of the web for dodgy adverts I came up with a couple of categories that are broad, and ambiguous, and that I could find images for relatively quickly.
